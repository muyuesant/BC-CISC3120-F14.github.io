Using Eclipse (Outside of Class)

There are two places you might want to use Eclipse outside of class: on a machine you control, and on a College computer. In the first case, you can just install Eclipse (as explained below), and it should work easily.

In the second case (College labs), here's what I've been advised: in order to maximize your flexibility (i.e. to be able to work at any lab computer), you should install Eclipse on a USB drive, and run it from there. This has worked for simple programs; I'm relying on you to tell me if you have any problems.

Installing Eclipse

Installing Eclipse is fairly straightforward: download and unzip; put it wherever you want (hard drive, USB drive, etc). If you have trouble with the unzip, try installing 7zip.

I strongly recommend installing the Eclipse version called "Eclipse IDE for Java EE Developers"—the Windows version is here. Once you've installed that, you need one more plugin. Under the "Help" menu, select "Install New Software...". On the window that pops up, click the "Add.." button and enter the URL http://download.eclipse.org/egit/github/updates . Once that's done, in the area labeled "type filter here", type github , and in the area below, you should see a single item, Eclipse GitHub integration with task focused interface . Select that item's checkbox, then click "Next" and do whatever is necessary to complete the install. Eclipse is ready to go.

Other Items for your USB Drive

If you're using Eclipse on USB, you should make sure that your Eclipse workspace is also on your USB drive. Once you select a place for it, you can tell Eclipse to make that your default workspace, so it will stop asking you each time you start.

Finally, you may want to be sure that Java 7 is available to you—it's hard to predict what will be on the College machines. (You may not need to do this; if you don't have any trouble without this step, you can skip it.) To do this, download the Java 7 JDK (currently at http://download.oracle.com/otn-pub/java/java_ee_sdk/7u3/java_ee_sdk-7u1.zip ). When you run the installer, make sure it installs on your USB drive. Record the folder name (probably something like e:\java\jdk1.7.3\ ).

One last step: you need to be able to tell the operating system where to find this version of Java, so you'll write a very short script. Using Notepad, create a file in the root directory of your USB drive called something like javaUSB.bat (the name doesn't really matter, but it needs to end in .bat ). In that file, type one line:

set Path=java\jdk1.7.3\bin;%Path%

Note: don't use a drive letter, but otherwise make sure the path matches the name of the folder where you installed Java.

In order to run Eclipse, and make sure that you have the right version of Java, open a command window and issue three commands:

E:
javaUSB
eclipse\eclipse.exe

Again, the drive letter and particular paths may vary depending on how you've set up your USB drive.
